Mini Project - Payment Manager Tool
Ex# 4: Mini Project One Requirements
  • March 22, 2022
      • Original Requirement
    • The first component will accept new payments. Data to capture will include customer name, price and their card number.
    • The second component will list the payments details.
    • Column to be displayed are as follows
      1. S.No: Dynamic row ID.
      2. Name: Name of the customer.
      3. Price: Price paid.
      4. Card Number: Card number used to make this payment.
      5. Operations: User can edit or delete this record using edit or delete button.
    • Payment details must be tracked with the help of a service.
    • Display the count of the total payments made.
  • March 24, 2022
      • Updated Requirement - 1
    • Search payment data by the column name using a custom pipe. Price column will have to match the exact number but the other columns can any match of the search value.
    • Mask the credit card number on display using a custom pipe.
    • Display 'Visa' image if the card number starts with 50 or above and 'Master' otherwise. Minimum search critiera is 2 digits.
  • March 25, 2022
      • Updated Requirement - 2
    • Custom Directive - Display hyphen after every 4 digits while displaying the card number.
    • Custom Pipe - While displaying the name column, the first letter must always start with capital.
  • March 28, 2022
      • Updated Requirement - 3
    • Move the search UI controls from 'List Payment' to it's parent component. This has to be done using @Input for the 'List Payment' to receive the criteria from it's parent.
  • March 30, 2022
      • Updated Requirement - 4
    • Apply template driven form validations.
